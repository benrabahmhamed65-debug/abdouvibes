Week 1 — The Foundation

Set up your environment and write your first code

Welcome to your first week! This week is all about getting set up and writing your very first lines of code. Don't worry about understanding everything perfectly — the goal is to get familiar with the tools, see code run, and feel the excitement of making something work.

You'll install VS Code (the most popular code editor in the world), learn what variables and functions actually are, and write your first real JavaScript program. By the end of this week, you should have a working "Hello World" and understand how a browser reads code.

Week 2 — Meet Your AI Partner

Learn to prompt AI and build your first AI-assisted project

This week you meet your most powerful tool: AI. You'll set up GitHub Copilot or start using Claude/ChatGPT directly, and you'll learn the single most important meta-skill of vibe coding — how to write a great prompt.

A great prompt has four parts: Context (what are you building?), Constraints (what language/framework?), Example (here's what I mean...), and Ask (what exactly do you want?). By the end of this week, you'll build a to-do app almost entirely with AI help — then read every line it gave you.

prompt_template.txt
1// Great Prompt Formula:
2CONTEXT: I'm building a to-do list app in vanilla JavaScript.
3STACK: HTML, CSS, and plain JS. No frameworks.
4EXAMPLE: When the user types a task and hits Enter, it appears
5 in a list below with a checkbox to mark it done.
6ASK: Write the addTask() function and the HTML input field.
7 
8// vs. a bad prompt:
9BAD: "make a todo app"
10// Too vague → mediocre output

Week 3 — Web Dev Fundamentals

Master HTML, CSS, and the DOM — the building blocks of every website

This week you go deep on the foundations of the web. HTML gives your page structure (headings, paragraphs, buttons, forms). CSS gives it style (colors, fonts, layout, animations). JavaScript makes it alive (respond to clicks, update content, animate things).

Together these three are called the DOM stack — Document Object Model. Learning how they interact is the single most valuable thing a web developer can know. This week you'll build a full personal portfolio page using AI and your own customizations.

dom_basics.js
1// The DOM — how JS talks to your HTML

Week 4 — APIs & Real Data

Connect your app to the real world with live data

This is where your apps stop being static and start being alive. An API (Application Programming Interface) is a way for your code to talk to other services — weather, maps, sports scores, cryptocurrency, AI, and thousands more.

You'll learn how to use fetch() to make HTTP requests, how to handle async/await (because data takes time to arrive), and how to display real data in your UI. By the end of this week, you'll have built a real weather app or movie search app that talks to a live server.

fetch_weather.js
1// Fetching real weather data from an API
2async function getWeather(city) {
3  const url = `https://wttr.in/${city}?format=j1`;
4  const response = await fetch(url); // wait for data
5  const data = await response.json(); // parse JSON
6  const temp = data.current_condition[0].temp_C;
7  return `${city}: ${temp}°C`;
8}
9 
10getWeather("Paris").then(result => console.log(result));

Week 5 — React Fundamentals

Build component-based UIs with the world's most popular framework

React is the most widely used JavaScript framework in the world. It lets you build UIs from components — small, reusable pieces of UI that manage their own data and update automatically when that data changes. Think of components like LEGO bricks for building interfaces.

The key concept is state — data that lives inside a component. When state changes, React automatically re-renders the UI to reflect it. This "reactivity" is what makes React so powerful. With AI, you can generate entire component trees in minutes and focus on the logic and user experience.

Counter.jsx
1import { useState } from "react";
2 
3// A simple React component with state
4export default function Counter() {
5  const [count, setCount] = useState(0);
6 
7  return (
8    <div>
9      <h2>Count: {count}</h2>
10      <button onClick={() => setCount(count + 1)}>
11        Click me!
12      </button>
13    </div>
14  );
15}

✅ Week 5 Tasks:

Set up a React project using: npm create vite@latest my-app -- --template react
Build the Counter component above and make it work in your browser
Understand useState: build a color picker that changes the page background
Learn useEffect: fetch weather data when the component loads (combine week 4 + 5!)
Build a small React app with at least 3 components (e.g. Header, Card, Footer)

Week 6 — Ship Your Project

Build something real, deploy it live, and share it with the world

This is it — the week everything comes together. You'll pick a project idea that genuinely excites you, build it using everything from weeks 1–5, and deploy it live on the internet for anyone to use. This is not a tutorial project — it's your own creation.

Deploying is easier than ever. Vercel and Netlify let you push your code and have it live in 60 seconds — for free. You'll also create a GitHub repository for your project, which serves as your public portfolio. By end of this week, you are no longer "learning to code." You are a vibe coder who ships.
